# Chp3-DevOps Concepts and Practices
## Build Automation
The automation of the process of preparing code for deployment to a live environment.

Benefits:
* Fast
* Consistent
* Repeatable
* Portable
* Reliable

## Continuous Integration (CI)
The practice of frequently merging code changes done by developers

Benefits:
* Early detection
* Eliminate the scramble
* Frequent releases
* Continuous testing
* good coding practices

## Continuous Delivery and Continuous Deployment (CD)
The practice of continuously maintaining code in a deployable state and frequently deploying small code changes to production

Benefits:
* Faster time-to-market
* Fewer problems caused by deployment process
* Low risk
* Reliable rollbacks
* Fearless deployments

## Infrastructure as Code (IaC)
Manage and provision infrastructure through code and automation

With IaC, instead of doing things manually, you use automation and code to create and change:
* Servers
* Instances
* Environments
* Containers
* Other infrastructure

Benefits:
* Consistency
* Reusability
* Scalability
* Self-documenting
* Simplify the complexity

## Configuration Management
Maintaining and changing the state of pieces of infrastructure in a consistent, maintainable and stable way

Benefits:
* Save time
* Insight
* Maintainability
* Less configuration drift

## Orchestration
Automation that supports processes and workflows, such as provisioning resources

Benefits:
* Scalability
* Stability
* Save time
* Self-service
* Granular insight into resource usage

## Microservices
A microservice architecture breaks an application up into a collection of small, loosely-coupled services

Benefits:
* Modularity
* Technological Flexibility
* Optimized Scalability
